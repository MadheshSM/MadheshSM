### How I Build

```
  DATA IN  ──►  clean · validate · structure
                        │
                        ▼
              choose architecture for the problem
              (not for the résumé)
                        │
                        ▼
              train · evaluate · explain
                        │
                        ▼
  SHIP  ────►  monitored · versioned · maintainable
```

Four things I don't compromise on:

| | |
|:--|:--|
| **Scalable** | Designed for growth from the first commit |
| **Efficient** | No wasted compute. No overengineered solutions |
| **Explainable** | Predictions must justify themselves |
| **Impact-driven** | Shipped to solve something. Not to showcase something |
